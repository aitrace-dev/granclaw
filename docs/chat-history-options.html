Chat History — Architecture Options

How should agent-brother store messages so agents remember across channels and the UI can display history?

Option A
Session per conversation

✓Each channel has its own Claude context window — no cross-talk
✓Fast — no extra tokens injected per request
✗Claude sessions expire — agent loses memory cold on restart
✗Can't move a conversation between channels
Option B
Inject history, no sessions
%%{init: {'theme': 'dark', 'themeVariables': {'background': '#1e1e1e', 'primaryColor': '#0ea5e9', 'primaryTextColor': '#e8e8e8', 'primaryBorderColor': '#0ea5e9', 'lineColor': '#555', 'secondaryColor': '#2a2a2a', 'tertiaryColor': '#1e1e1e', 'edgeLabelBackground': '#1e1e1e', 'fontSize': '12px'}}}%% flowchart TD WA["📱 WhatsApp"] -->|msg| B LI["💼 LinkedIn"] -->|msg| B UI["🖥 Dashboard UI"] -->|msg| B B["Backend\nOrchestrator"] B -->|fetch last N msgs| MG[("MongoDB\nmessages")] MG -->|history| B B -->|prompt = history + msg| AG["🤖 Agent\nRunner"] AG -->|spawns fresh| CL["claude CLI\n-p 'prev: ...\nuser: ...'"] AG -->|save reply| MG MG -->|display| UI
✓Full control over what agent sees — no session expiry surprises
✓Conversation can move freely across channels
✗Extra tokens on every request — slower and more expensive
✗Context window fills up on long conversations
Option C — Recommended
Hybrid: resume + fallback inject
%%{init: {'theme': 'dark', 'themeVariables': {'background': '#1e1e1e', 'primaryColor': '#10b981', 'primaryTextColor': '#e8e8e8', 'primaryBorderColor': '#10b981', 'lineColor': '#555', 'secondaryColor': '#2a2a2a', 'tertiaryColor': '#1e1e1e', 'edgeLabelBackground': '#1e1e1e', 'fontSize': '12px'}}}%% flowchart TD WA["📱 WhatsApp"] --> B LI["💼 LinkedIn"] --> B UI["🖥 Dashboard UI"] --> B B["Backend\nOrchestrator"] B -->|save msg| MG[("MongoDB\nmessages")] B -->|run| AG["🤖 Agent\nRunner"] AG -->|happy path| CL1["claude CLI\n--resume session_id"] CL1 -->|exit 1 / expired| FB["⚠ Session expired\nfallback"] FB -->|fetch last 20 msgs| MG MG --> FB FB -->|inject as context| CL2["claude CLI\nfresh + history"] CL1 -->|reply| B CL2 -->|reply + new session_id| B B -->|save reply| MG MG -->|display| UI
✓Fast on happy path (--resume, no extra tokens)
✓Survives session expiry — injects history automatically
✓Full message log in MongoDB for UI + cross-channel queries
✗Slightly more complex runner logic (already partially there)

Recommended data model — Option C

MongoDB messages collection + SQLite agent_sessions per (agent, channel, conversation)
